# Operations OS

## 目的

日次・週次・月次・四半期・年次の運営ルーティンを定義する。
一人創業者が複数事業を動かしながら、重要なことを見落とさないための仕組み。

## 何を保存するか

- **日次ルーティン** — 毎日やること・確認すること
- **週次レビュー手順** — 週の振り返りと翌週の計画
- **月次レビュー手順** — 財務・KPI・プロダクト・ポートフォリオの確認
- **四半期レビュー手順** — 戦略の見直し・原則の更新
- **年次レビュー手順** — 長期ビジョンの更新・FOUNDER_PHILOSOPHYの見直し
- **インシデント対応手順** — 障害・問題発生時の動き方

## ルーティンの設計原則

1. **必要最小限に保つ** — ルーティンが重くなると続かない
2. **チェックリストで迷いをなくす** — 毎回考え直さない
3. **時間を決める** — 「いつかやる」は「やらない」
4. **記録する** — 振り返りができるように

## 週次レビュー（基本構成）

```
毎週月曜 30分
1. 先週の主要KPIを確認 (5分)
2. 進行中のタスクの状況確認 (10分)
3. 今週の最重要タスク3つを決める (10分)
4. ブロッカー・懸念事項を書き出す (5分)
```
