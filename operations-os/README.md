# Operations OS

## 目的

日次・週次・月次・四半期・年次の運営ルーティンを定義する。
一人創業者が複数事業を動かしながら、重要なことを見落とさないための仕組み。

**このOSが答える問い:**
- 毎日・毎週・毎月やるべきことは何か
- どのタイミングで何を確認し、何を判断するか
- インシデントが起きたときどう動くか
- 年次でOSと事業をどう見直すか

---

## 責務

| 責務 | 内容 |
|------|------|
| 日次ルーティン | 毎日確認すること・やること |
| 週次レビュー手順 | 週の振り返りと翌週の計画 |
| 月次レビュー手順 | 財務・KPI・プロダクトの確認 |
| 四半期レビュー手順 | 戦略の見直し・OS原則の更新 |
| 年次レビュー手順 | 長期ビジョンの更新・FOUNDER_PHILOSOPHYの見直し |
| インシデント対応手順 | 障害・問題発生時の動き方と事後レポート |
| OS更新ルーティン | IPPOの知見をFounder OSに反映するサイクル |

---

## ルーティンの設計原則

1. **必要最小限に保つ** — ルーティンが重くなると続かない
2. **チェックリストで迷いをなくす** — 毎回考え直さない
3. **時間を決める** — 「いつかやる」は「やらない」
4. **記録する** — 振り返りができるように

| 頻度 | 所要時間（目安） | 主な内容 |
|------|--------------|---------|

| 月次 | 2時間 | 財務・KPI・CS・ポートフォリオ全体レビュー |
| 四半期 | 半日 | 戦略見直し・OS更新・ROADMAP確認 |
| 年次 | 1日 | FOUNDER_PHILOSOPHY見直し・年間計画 |
