# Brand OS

## 目的

複数プロダクトにまたがるブランドの一貫性を保つための基準集。
トーン・ビジュアル・メッセージングの「迷わない基準」を定義する。

**このOSが答える問い:**
- どんな言葉を使い、どんな言葉を使わないか
- プロダクトの「声」はどんな質感か
- ビジュアルの優先順位は何か
- 複数プロダクトで共通・差分があるブランド要素は何か

---

## 責務

| 責務 | 内容 |
|------|------|
| ブランドボイス | 言葉のトーン・口調・禁止表現 |
| ビジュアルアイデンティティ原則 | 色・タイポ・スペーシングの思想（実装はプロダクト側） |
| ネーミング規約 | プロダクト名・機能名の命名ルール |
| コピーライティングガイド | LP・App Store・SNS別の書き方 |
| ブランドの境界 | 何をブランドとして扱うか・扱わないか |
| プロダクト間の差分管理 | 共通ブランドと個別ブランドの分離 |

---

## 将来追加予定

- `BRAND_VOICE.md` — 声のトーン・禁止表現・文例集
- `VISUAL_IDENTITY_PRINCIPLES.md` — 色・タイポ・余白の原則（実装コードは各プロダクト）
- `NAMING_CONVENTIONS.md` — プロダクト・機能・URLの命名ルール
- `COPYWRITING_GUIDE.md` — App Store / LP / SNS別の書き方
- `PRODUCT_BRAND_PROFILES.md` — プロダクト別のブランド個性の定義
