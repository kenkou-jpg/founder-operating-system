# Brand OS

## 目的

複数プロダクトにまたがるブランドの一貫性を保つための基準集。
トーン・ビジュアル・メッセージングの「迷わない基準」を定義する。

## 何を保存するか

- **ブランドボイス** — 言葉のトーン・口調・禁止表現
- **ビジュアルアイデンティティ原則** — 色・タイポ・スペーシングの思想
- **ネーミング規約** — プロダクト名・機能名・ファイル名の命名ルール
- **コピーライティングガイド** — LP・App Store・SNS別の書き方
- **ブランドの境界** — 何をブランドとして扱うか・扱わないか

## ブランド原則

1. **誠実さが最初** — 過剰な約束・誇大表現は使わない
2. **シンプルに届ける** — 専門用語を避け、中学生でも理解できる言葉を選ぶ
3. **一貫したトーン** — プロダクトが変わっても、声の質は同じ
4. **日本語ファースト** — 現時点の主要市場に最適化

## プロダクト別ブランド定義

| プロダクト | ブランドファイル |
|-----------|--------------|
| ippo | `portfolio-os/IPPO.md` |
| AgriPath | `portfolio-os/AGRIPATH.md` |
| Fasting App | `portfolio-os/FASTING_APP.md` |

## 今後の再利用計画

- 新プロダクト追加時に `portfolio-os/` にブランドドキュメントを追加
- ブランドボイスはコピーライティングAIへのプロンプトとしても活用
