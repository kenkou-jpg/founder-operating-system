# Research OS

## 目的

市場調査・競合分析・ユーザーリサーチの手順と蓄積を管理する。
「知ったこと」を事業判断につながる形で保存する場所。

## 何を保存するか

- **市場調査フレームワーク** — TAM/SAM/SOM の定義手順
- **競合分析テンプレート** — 比較軸の標準化
- **ユーザーインタビュー手順** — 質問設計・録音・メモのルール
- **インサイトアーカイブ** — 発見を構造化して保存
- **情報源リスト** — 信頼できるデータソース・論文・レポートの一覧
- **調査ルーティン** — 月次でどの情報をウォッチするか

## リサーチ原則

1. **一次情報を優先** — ユーザーの声 > 調査レポート > 憶測
2. **バイアスを記録する** — 調査者のバイアスを明示する
3. **日付を必ず記録** — 市場情報は鮮度が命
4. **「発見」と「解釈」を分ける** — 事実と自分の推論を区別して記録

## 今後の再利用計画

- **農業市場（AgriPath）** — 農家インタビューの手法・質問設計を蓄積
- **ヘルスケア市場（Fasting App）** — 競合（MyFitnessPal等）の定期モニタリング
- **ウォーキング市場（ippo）** — ユーザーモチベーション研究の蓄積
