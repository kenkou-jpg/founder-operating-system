# Research OS

## 目的

市場調査・競合分析・ユーザーリサーチ・学術調査の手順と蓄積を管理する。
「知ったこと」を事業判断につながる形で保存する場所。

**Imaging Agriculture** のような研究事業では、このOSの誠実さが倫理の問題になる。

**このOSが答える問い:**
- 市場規模をどう調べ、どう見積もるか
- ユーザーインタビューをどう設計・実施・活用するか
- 競合をどう分析し、どう活用するか
- 学術論文・研究データをどう扱うか

---

## 責務

| 責務 | 内容 |
|------|------|
| 市場調査フレームワーク | TAM/SAM/SOMの定義手順 |
| 競合分析テンプレート | 比較軸の標準化 |
| ユーザーインタビュー手順 | 質問設計・記録・分析のルール |
| インサイトアーカイブ | 発見を構造化して保存する方法 |
| 情報源評価基準 | 信頼できる情報源・疑うべき情報源の判断軸 |
| 調査ルーティン | 月次でどの情報をウォッチするか |
| 学術調査倫理 | 研究データの扱い・引用・再利用のルール |

## 他OSとの関係

| OS | 関係 |
|----|------|
| `business-os/` | 市場調査の結果はGTM・価格設計の根拠として使用 |
| `knowledge-os/` | リサーチで得た知見はknowledge-osに統合 |
| `analytics-os/` | 定量データの解釈・活用方針を連携 |
| `portfolio-os/` | 各プロダクトの市場調査はportfolio-osのプロファイルにリンク |
