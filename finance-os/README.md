# Finance OS

## 目的

財務管理・コスト構造・投資判断の基準を定義する。
一人創業者が財務的に持続可能な複数事業を運営するための手順書。

**このOSが答える問い:**
- 毎月どこにいくら使っているか、それは適切か
- いつプロダクトへの投資を増やし、減らすか
- ランウェイはどのくらいあるか
- 何にお金をかけて、何にかけないか

---

## 責務

| 責務 | 内容 |
|------|------|
| コスト管理方針 | 固定費・変動費の管理ルール、上限基準 |
| 投資判断フレームワーク | 何にお金をかけるか・かけないかの軸 |
| 収益目標の設定方法 | ARR/MRR目標の設定と追跡 |
| 財務レビュー手順 | 月次・四半期・年次のレビュープロセス |
| ランウェイ管理 | キャッシュフロー計算と最低ランウェイ基準 |
| 税務・会計の準備手順 | 経費計上・確定申告の準備チェックリスト |
| ツールコスト管理 | SaaS・インフラコストの定期見直し手順 |

## 他OSとの関係

| OS | 関係 |
|----|------|
| `business-os/` | 収益目標・価格設計はbusiness-osと整合 |
| `analytics-os/` | MRR・ARR・LTVの数値はanalytics-osのKPI定義に従う |
| `operations-os/` | 月次財務レビューはoperations-osのルーティンに組み込む |
| `portfolio-os/` | ポートフォリオ全体の財務サマリーはPORTFOLIO_KPIに反映 |
| `automation-os/` | 財務レポートの自動化ルールはautomation-osに記録 |
