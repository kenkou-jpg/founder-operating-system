# Automation OS

## 目的

繰り返し作業の自動化・CI/CDパイプライン・Workflow設計の記録と標準を管理する。
一人創業者が複数事業を動かすための「時間を増やすOS」。

**OPERATING_PRINCIPLESの「Automation First」を実装するOS。**

**このOSが答える問い:**
- 何を自動化するか（しないか）
- 自動化が壊れたときどう検知するか
- GitHub Actionsをどう標準化するか
- どのツールを使い、どう連携させるか

---

## 責務

| 責務 | 内容 |
|------|------|
| 自動化の判断基準 | 何を自動化するか・しないかの軸 |
| GitHub Actions 標準構成 | CI/CDのベストプラクティスと共通ワークフロー |
| 定期タスクの一覧 | cron・スケジューラーで動くもの一覧 |
| 通知設計 | どのイベントで・何を通知するか |
| 外部サービス連携の管理 | Webhook・API連携の記録と更新ルール |
| 自動化の監視とアラート | 自動化が壊れたときの検知方法 |
| 自動化の廃止ルール | 使われなくなった自動化の削除手順 |

---

## 自動化の判断基準（3×3ルール）

1. **同じ作業を3回やった** → 自動化を検討
2. **自動化のコストが3週間で回収できる** → 自動化する
3. **自動化が壊れたとき3分で気づける** → 自動化を維持
