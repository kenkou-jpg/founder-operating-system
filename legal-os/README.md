# Legal OS

## 目的

全プロダクト共通の法務方針・利用規約ひな形・コンプライアンス基準を管理する。
法務コストを最小化しつつ、ユーザー保護と事業保護を両立する。

**このOSが答える問い:**
- 利用規約・プライバシーポリシーをどう書くか
- ヘルスケア・農業等のセンシティブ領域での法的リスクは何か
- 外部サービスの利用条件で気をつけることは何か
- データ収集・保存・削除のルールをどう設計するか

> **注意:** このOSは法律の専門家による確認を前提とするひな形・チェックリスト集です。
> 実際の法的文書は弁護士レビューを経て公開してください。

---

## 責務

| 責務 | 内容 |
|------|------|
| 利用規約ひな形 | プロダクト別にカスタマイズできる基本テンプレート |
| プライバシーポリシーひな形 | データ収集・利用・保護の標準文書 |
| 特定商取引法表記 | EC・サブスクリプションサービス向け |
| 免責事項テンプレート | ヘルスケア・農業等のセンシティブ領域向け |
| 外部サービス利用条件チェック | Supabase・Expo・Apple・Googleの規約注意点 |
| 著作権・ライセンス方針 | OSSライセンスの扱い・自社著作物の保護 |
| 規約変更手順 | 変更の告知・同意取得のルール |

## 他OSとの関係

| OS | 関係 |
|----|------|
| `development-os/` | セキュリティ基準はlegal-osのデータ保護方針と整合させる |
