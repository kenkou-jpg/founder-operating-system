# Customer Success OS

## 目的

ユーザーサポート・オンボーディング・フィードバック収集の標準手順を定義する。
一人で複数プロダクトのCSを回すための効率的なシステム。

**このOSが答える問い:**
- 問い合わせにどう・いつ・どんなトーンで返すか
- ユーザーにどうオンボーディングするか
- フィードバックをどう収集し、どう活用するか
- 解約ユーザーからどう学ぶか

---

## 責務

| 責務 | 内容 |
|------|------|
| サポート対応の標準 | 返信スピード・トーン・エスカレーション判断 |
| FAQ管理のルール | よくある質問の蓄積・公開・更新手順 |
| オンボーディング設計原則 | 初日・初週・初月の体験設計の考え方 |
| フィードバック収集の手順 | アプリ内・メール・SNSからの収集と整理 |
| チャーン分析の手順 | 解約ユーザーへの対応と学習プロセス |
| ユーザーコミュニティ方針 | SNS・コミュニティの運営方針 |

---

## 将来追加予定

- `SUPPORT_STANDARDS.md` — 返信SLA・トーン・禁止表現・エスカレーション
- `ONBOARDING_PRINCIPLES.md` — 初期体験設計の原則と測定方法
- `FEEDBACK_MANAGEMENT.md` — フィードバック収集・分類・活用の手順
- `CHURN_ANALYSIS.md` — 解約分析の手順と学習テンプレート
- `response-templates/` — 状況別の返信テンプレート集
- `FAQ_MANAGEMENT.md` — FAQ作成・更新・公開のルール

## 他OSとの関係

| OS | 関係 |
|----|------|
| `brand-os/` | CS対応のトーン・言葉遣いはBrand Voiceに従う |
| `analytics-os/` | CSメトリクス（返信時間・満足度・解約率）はKPI定義に従う |
| `legal-os/` | 法的問い合わせの対応手順はlegal-osに記載 |
| `knowledge-os/` | ユーザーから学んだことはknowledge-osに蓄積 |
| `research-os/` | フィードバックはユーザーリサーチの一次情報として活用 |
| `operations-os/` | CSレビューはoperations-osの週次ルーティンに含める |
