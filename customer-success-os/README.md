# Customer Success OS

## 目的

ユーザーサポート・オンボーディング・フィードバック収集の標準手順を定義する。
一人で複数プロダクトのCSを回すための効率的なシステム。

## 何を保存するか

- **サポート対応の標準** — 返信スピード・トーン・エスカレーション判断
- **FAQ管理のルール** — よくある質問の蓄積・公開・更新手順
- **オンボーディング設計原則** — 初日・初週・初月の体験設計
- **フィードバック収集の手順** — アプリ内・メール・SNSからの収集と整理
- **チャーン分析の手順** — 解約ユーザーへの対応と学習
- **ユーザーコミュニティの運営方針** — SNS・コミュニティの扱い

## CS原則

1. **24時間以内に返信** — 問い合わせには24時間以内に初回返信
2. **テンプレートより誠実さ** — コピペ返信でなく、問題を理解して返す
3. **フィードバックは資産** — すべてのフィードバックを記録・分類・活用
4. **解約は情報** — 怒らず、学ぶ

## 今後の再利用計画
