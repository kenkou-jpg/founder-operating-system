# Business OS

## 目的

事業戦略・収益モデル・GTM（Go-to-Market）・価格設計の共通思想と手順を定義する。
各プロダクトのビジネス判断の土台となるOS。

**このOSが答える問い:**
- 何を収益化するか（しないか）
- どこに・どうやってユーザーを連れてくるか
- 価格をどう設定し、変更するか
- プロダクトをいつ終了するか

---

## 責務

| 責務 | 内容 |
|------|------|
| 事業モデルの評価基準 | 収益モデルの選定・評価の軸 |
| GTMフレームワーク | 新プロダクトの市場投入手順 |
| 価格設計原則 | 価格帯・プラン構成・値上げのルール |
| PMF基準 | プロダクト-マーケットフィットをどう定義・測定するか |
| 撤退基準 | いつプロダクトを終了するかの判断軸 |
| 競合分析フレームワーク | 市場をどう見るかの視点と手順 |

## 他OSとの関係

| OS | 関係 |
|----|------|
| `analytics-os/` | KPIで事業判断を行う。KPI定義はanalytics-osが管理 |
| `research-os/` | 市場調査・競合分析の一次情報はresearch-osから |
| `finance-os/` | 収益目標・コスト構造はfinance-osと整合 |
| `brand-os/` | GTMのメッセージングはbrand-osの基準に従う |
| `portfolio-os/` | 各プロダクトの事業戦略はportfolio-osのプロファイルにリンク |
