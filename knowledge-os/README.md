# Knowledge OS

## 目的

学習・インサイト・ナレッジキャプチャのシステムを管理する。
読んだこと・学んだこと・気づいたことを、事業に活用できる形で保存する。

**OPERATING_PRINCIPLESの「Knowledge Compounds」を実装するOS。**

**このOSが答える問い:**
- 学んだことをどうキャプチャし、どこに保存するか
- 積み上がったナレッジをどう活用するか
- 一つのプロダクトで得た知見をどう他に展開するか
- AIからの学びと自分の判断軸をどう統合するか

---

## 責務

| 責務 | 内容 |
|------|------|
| キャプチャシステム | 学んだことをすぐ書く仕組みと手順 |
| パターンライブラリ | 繰り返し登場する問題とその解決パターン |
| 失敗アーカイブ | 失敗・間違いから学んだことの構造化記録 |
| 業界知識の蓄積 | 農業・ヘルスケア・ウォーキング等の専門知識 |
| ツール評価記録 | 使ったツールの評価・選定理由・代替案 |
| OSへのフィードバックループ | 知見をFounder OSへ反映するプロセス |

---

## ナレッジキャプチャの原則

1. **読んだらすぐ書く** — 24時間以内に要点をメモ
2. **自分の言葉で書く** — コピペは知識にならない
3. **「だから何か」まで書く** — 事業への示唆まで落とす
4. **定期的に見直す** — 溜めるだけでなく活用する
