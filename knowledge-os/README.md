# Knowledge OS

## 目的

学習・インサイト・ナレッジキャプチャのシステムを管理する。
読んだこと・学んだこと・気づいたことを、事業に活用できる形で保存する。

## 何を保存するか

- **学習の記録** — 本・記事・ポッドキャストから得た重要な知見
- **パターンライブラリ** — 繰り返し登場する問題とその解決パターン
- **失敗から学んだこと** — プロジェクト・判断の振り返り
- **業界知識の蓄積** — 農業・ヘルスケア・ウォーキング等の専門知識
- **ツール評価記録** — 使ったツールの感想・選定理由・代替案

## ナレッジキャプチャの原則

1. **読んだらすぐ書く** — 24時間以内に要点をメモ
2. **自分の言葉で書く** — コピペは知識にならない
3. **「だから何か」まで書く** — 事業への示唆まで落とす
4. **定期的に見直す** — 蓄積するだけでなく活用する

## 今後の再利用計画

- `research-os/` と連携してインサイトを事業判断に繋げる
- パターンライブラリをAIプロンプトとして活用
