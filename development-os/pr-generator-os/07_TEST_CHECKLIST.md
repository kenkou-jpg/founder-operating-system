# 07 — Test Checklist

> PRごとのテスト確認テンプレート。
> 実装前（何をテストするか計画）と実装後（テストがPASSしているか確認）の両方で使う。

---

## テストの原則

1. **テストなしのPRはDefinition of Doneを満たさない**
2. **既存テストを壊してはいけない（Regression禁止）**
3. **テストは実装と同時に書く（後回し禁止）**
4. **テストは独立して実行できる（他テストへの依存禁止）**
5. **テストはドキュメントでもある（テスト名で仕様がわかるように）**

---

## テスト種別チェックリスト

### Unit Test

- [ ] **正常系テスト**
  - 期待通りの入力で期待通りの出力が返るか
  - すべての主要なパスをカバーしているか

- [ ] **異常系テスト**
  - 不正な入力に対して適切なエラーが返るか
  - null / undefined / 空文字 / 空配列の処理

- [ ] **境界値テスト**
  - 最小値・最大値・境界付近での動作
  - オフバイワンエラーがないか

### Domain Test

- [ ] **Domain Logic テスト**
  - Domainルールが正しく適用されるか
  - ValueObjectの同値性テスト
  - EntityのLifecycleテスト

- [ ] **Rule Engine テスト（該当する場合）**
  - すべてのRule条件分岐をカバー
  - 優先順位が正しいか
  - コンフリクトするRuleの扱い

### Repository Test

- [ ] **Repository Unit テスト（該当する場合）**
  - Interface準拠の確認
  - CRUD操作のテスト

- [ ] **Adapter テスト（該当する場合）**
  - 外部サービスとのマッピングが正しいか
  - エラーハンドリングのテスト

### Integration Test

- [ ] **Integration テスト**
  - 複数レイヤーを跨いだ統合テスト
  - 実際のサービスを使ったEnd-to-Endに近いテスト

### API Test（API PRの場合）

- [ ] **APIエンドポイントテスト**
  - 正常レスポンスのテスト
  - 認証エラーのテスト
  - バリデーションエラーのテスト
  - 存在しないリソースへのアクセス（404）

### Architecture Guard Test

- [ ] **レイヤー境界テスト**
  - 禁止された依存方向が存在しないことを自動検証
  - 新しいモジュールに対してGuardを追加したか

### Regression Test

- [ ] **既存テストの PASS 確認**
  - 本PR実装後も既存テストがすべてPASSするか
  - `git stash` 後に `npm test` を実行して確認

- [ ] **既存機能への影響なし**
  - 変更が意図しない副作用を起こしていないか

### Failure Test

- [ ] **障害シナリオテスト**
  - ネットワーク切断時の動作
  - DBエラー時の動作
  - タイムアウト時の動作

### Performance Test（必要な場合）

- [ ] **パフォーマンステスト**
  - 大量データ処理の速度確認
  - メモリリーク確認
  - レスポンスタイムが許容範囲内か

### Append-Only Test（Event Sourcing PRの場合）

- [ ] **追記専用テスト**
  - Eventが削除・更新されていないことの確認
  - Eventの順序が保証されているか

### Migration Test（Migration PRの場合）

- [ ] **Migration Up テスト**
  - Migrationが正常に実行されるか
  - データが正しく変換されるか

- [ ] **Migration Down テスト（Rollback）**
  - ロールバックが正常に動作するか
  - データが元の状態に戻るか

### Build Test

- [ ] **ビルドテスト**
  - `npm run build` / `expo build` が成功するか
  - 型エラーがないか（TypeScript）
  - Lintエラーがないか

## テスト実行結果（実装後に記入）

```
テスト実行結果
--------------
追加テスト数: X件
総テスト数（既存 + 新規）: X件
PASS: X件
FAIL: X件
SKIP: X件

既存テストのRegression: あり / なし
（あり の場合は詳細を記録）

Build: PASS / FAIL
型チェック: PASS / FAIL
Lint: PASS / FAIL

→ テスト結果: PASS / STOP
```

**FAIL または Regression がある場合は、解消するまでPRを進めない。**
