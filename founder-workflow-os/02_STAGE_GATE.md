# 02 — Stage Gate

> 各ステージの Go / No-Go 条件を定義する。
> ここで「No-Go」になった場合、前のステージに戻って解決してから進む。
> 気持ちで進まない。条件で進む。

---

## Stage Gate の使い方

1. 次のステージに進む前に、このファイルの対応するゲートを確認する
2. すべての条件が ✅ になるまで次に進まない
3. どうしても進めない条件があれば、`governance/DECISION_LOG.md` に理由を記録してFounder判断

### Gate Sc2 — Next App 開始条件（最重要）

| 条件 | 確認 |
|-----|------|
| 現在の開発中プロダクト数が上限（2つ）以内 | ⬜ |
| 既存プロダクトの Retention が目標値を達成している | ⬜ |

→ **Go / No-Go**

**1つでも No の場合、Next App を始めない。**
これは OPERATING_PRINCIPLES #15 Portfolio Thinking の実装。

## No-Go 時の対応

| No-Go の理由 | 対応 |
|------------|------|
| 前ステージが未完了 | 前ステージに戻って完了させる |
| 市場・事業性に問題 | S01〜S03 をやり直すか、アイデアを棄却 |
| 品質基準未達 | 修正後に再チェック |
| Founder の健康・ペース問題 | スケジュールを見直す。無理に進めない |
| 法的リスクが解決しない | `legal-os/` + 専門家相談 |
